# 待处理问题

## 2026-06-01 agent-reach 调用 Twitter/X 搜索不稳定

### 背景

用户连续三次要求 agent-alpha 使用 `agent-reach` 去 Twitter/X 调研 Claude Code dynamic workflows。最新 session 日志显示，`agent-reach` skill 已安装，且后两次可以通过 `load_skill` 成功加载：

- `agent-alpha/home/.agents/skills/agent-reach/SKILL.md`
- `agent-alpha/.venv/Scripts/agent-reach.exe`
- `agent-alpha/.venv/Scripts/twitter.exe`

### 失败链路

1. 2026-06-01 11:16 会话：alpha 没有调用 `load_skill`，直接使用 `mcp__open-websearch__search`，没有真正走 `agent-reach`。
2. 2026-06-01 11:33 会话：alpha 成功加载 `agent-reach`，执行 `twitter search "Claude Code dynamic workflow subagent pipeline" -n 20`，但命令被 ESC 中断，返回 `Command interrupted by ESC`。
3. 2026-06-01 11:39 会话：alpha 成功加载 `agent-reach`，执行 `twitter search "Claude Code dynamic workflow subagent" -n 15`，但 300 秒内无 stdout/stderr，最终被 bash 工具超时杀掉，返回 `命令超时（>300秒）`。

### 初步判断

这不是 `agent-reach` skill 文档未加载，也不是 `twitter.exe` 不存在。问题更可能出在 `twitter-cli` 真正访问 X 搜索接口时卡住：

- X 搜索接口本身反爬、限流或风控较强；
- cookie 存在，但不代表当前请求一定能通过 X 风控；
- 当前网络访问 X 可能慢、不稳定或需要代理；
- `twitter-cli` 中间过程没有持续输出，导致 alpha 只能等待到 bash 工具 300 秒超时。

### 后续处理建议

1. 在使用 `agent-reach` 前，先让 alpha 执行 `agent-reach doctor` 或 `twitter status`，确认 Twitter 通道状态。
2. 对 `twitter search` 增加更短的外层超时策略，避免一次调用卡满 300 秒。
3. 如果 `twitter search` 失败，优先尝试 skill 文档建议的 `twitter user-posts`、`twitter tweet`、`twitter thread`，不要直接退回普通 web search。
4. 检查是否需要为 `twitter-cli` 配置代理，或更新 cookie。
5. 优化 alpha 的执行纪律：用户明确要求使用某个 skill 时，应先加载 skill 并按 skill 文档链路执行，失败时明确报告失败点。
