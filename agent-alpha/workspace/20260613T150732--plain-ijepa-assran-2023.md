## 这篇论文在干什么

这是 **Yann LeCun 的 JEPA 理念第一次在图像上真正落地**。

2022 年 LeCun 发了那篇宣言论文《A Path Towards Autonomous Machine Intelligence》，画了一张大蓝图。但蓝图只是蓝图——**没有人实现过**。

这篇就是"动手干"的结果。作者把 JEPA 思想实现在了图像上，取名叫 **I-JEPA**（Image-based Joint-Embedding Predictive Architecture），然后用它做了大量实验，证明了：

> **不靠数据增强、不靠像素重建，只用"在抽象空间里做预测"这一个思路，就能学到非常强的图像特征。**

---

## 问题在哪：两条老路都有毛病

在 I-JEPA 之前，自监督学习（不用标签自己学）有两大流派，各有各的毛病：

### 流派 1：不变性方法（Invariance-based）

**代表作：** DINO、SimCLR、BYOL、SwAV

**做法：** 把一张图片做各种"手脚"——随机裁剪、旋转、变色、模糊——制造出两张看起来不一样的"视图"（view），然后训练编码器把这两张视图编码成**一样的向量**。

```
原图 "猫" → 裁剪旋转 → 视图 A → 编码器 → [0.3, 0.7, ...]
原图 "猫" → 颜色抖动 → 视图 B → 编码器 → [0.3, 0.7, ...]  ← 一样！
```

**优点：** 学到的特征语义强，分类任务表现好。

**毛病：**
- 依赖**手工设计的数据增强**——裁剪多大、旋转多少度、颜色抖多狠，全是人调的
- 这些"手脚"引入了很强的**偏见**——比如你把图片裁得只剩猫头，那模型学到的就是"猫=猫头"，换一张猫躲在树后的图就傻了
- 用图像特制的增强（裁剪、变色）很难迁移到其他模态（音频、文本）

### 流派 2：生成式方法（Generative / Masked Image Modeling）

**代表作：** MAE（Masked Autoencoder）、BEiT、SimMIM

**做法：** 遮住图片的一部分，让模型**预测被遮住的像素**。

```
原图 "猫" → 遮住 75% → 编码器 → 解码器 → 猜出被遮的像素
```

**优点：** 不依赖手工数据增强，通用性强。

**毛病：**
- 模型在猜**每个像素的值**，浪费大量算力去预测"树叶纹理""背景花纹"这种对语义无关紧要的细节
- 学到的特征**语义级别低**，拿去做分类任务不如不变性方法

### I-JEPA 的定位：第三条路

I-JEPA 卡在两条路中间：

- **不像不变性方法**：不需要数据增强（不用制造多张视图）
- **不像生成式方法**：不预测像素（不在像素空间做重建）

而是：**遮住一块区域，预测这块区域在抽象空间里的表示。**

## I-JEPA：核心思路

### 一句话

> **给你的图片编码器一块"上下文"，让它猜出另一块区域"应该编码成什么向量"。**

### 三件东西

I-JEPA 架构里有三个角色：

```
         ┌──────────────────────────────────────────┐
         │               I-JEPA                      │
         │                                            │
         │   上下文编码器    预测器     目标编码器     │
         │  (context-encoder)  (predictor) (target-encoder) │
         │                                            │
         │  只看上下文块    猜目标块表示    编码目标块     │
         └──────────────────────────────────────────┘
```

### 具体怎么操作（四步走）

**第一步：选目标块（Targets）**

把一张图片切成很多小补丁（patch）。随机选 **4 个比较大的方块**作为"目标块"。

每个目标块的大小在图片面积的 **15%~20%** 之间——这个大小很关键，太小了学不到语义（比如只看一只猫耳朵），太大了又太简单。

```
┌─────────────────────┐
│                     │
│    ████  ████      │  ← 4 个目标块
│    ████  ████      │     每个占图片 15-20%
│                     │
└─────────────────────┘
```

**第二步：选上下文块（Context）**

再选**一个很大的方块**作为"上下文"。这个上下文块占图片的 **85%~100%**，然后**去掉和目标块重叠的部分**。

```
┌─────────────────────┐
│ ┌─────────────────┐ │
│ │ 上下文块 (超大)   │ │
│ │ 但和目标重叠的    │ │
│ │ 区域被挖掉了     │ │
│ └─────────────────┘ │
└─────────────────────┘
    剩下的就是"上下文"
```

**第三步：编码器和预测器工作**

- **上下文编码器**（一个 ViT）只看上下文块，输出一个抽象表示 sₓ
- **预测器**（一个轻量 ViT）接收 sₓ 和"目标块的位置信息"，输出它对目标块表示的**猜测** ŝ_y
- **目标编码器**（跟上下文编码器结构一样，但权重不同）看完整图片，输出目标块的真实表示 s_y

```
上下文块 → 上下文编码器 f → sₓ → 预测器 g → ŝ_y (猜测)
                                             对比 ↓
完整图片 → 目标编码器 f̄  → s_y (真实答案)
```

**第四步：算损失**

```
Loss = L2距离(ŝ_y, s_y)
     = 平均每个补丁的 (猜测值 - 真实值)²
```

预测得越准，损失越低。

### 关键设计细节

**① 目标编码器的权重怎么更新的？**

不是通过梯度下降学的，而是**指数移动平均（EMA）**：

```
目标编码器权重 = 0.996 × 旧权重 + 0.004 × 上下文编码器权重
```

即目标编码器是上下文编码器的**慢速移动版本**。这个设计源于 BYOL、DINO 等前人工作，防止表示塌缩（所有图片编码成同一个向量）。

**② 预测器必须窄（bottleneck）**

预测器的宽度（embedding dimension）固定为 **384**，不管上下文编码器有多大（ViT-B 是 768，ViT-L 是 1024，ViT-H 是 1280）。这个瓶颈设计**迫使**预测器不能直接复制粘贴——它必须在有限的容量里学出真正的预测能力。

实验证明：预测器用 384 维比用 1024 维表现更好。

**③ 目标块的大小必须适中**

太小（< 5% 图片）：学到的太局部、太像素级
太大（> 30% 图片）：预测太容易、学不到好特征

最佳区间：**15%~20%**，这个大小的块通常包含一个"有意义的东西"——猫头、车轮、窗户。

## Masking 策略的消融实验（核心发现）

这是论文最关键的实验部分。作者比较了四种 masking 策略：

| 策略 | 做法 | 1% ImageNet 精度 |
|------|------|:---:|
| **Multi-block（I-JEPA 用的）** | 1 个大上下文块预测 4 个中等目标块 | **54.2%** |
| Rasterized | 图片分 4 象限，1 个象限预测其他 3 个 | 15.5% |
| Block | 1 个目标块，剩余全是上下文 | 20.2% |
| Random | 随机遮 60% 补丁，剩的是上下文 | 17.6% |

**Multi-block 完胜。** 为什么？

- **Rasterized 问题：** 一个象限信息量太少，猜不出其他象限的内容
- **Block 问题：** 一个目标块太简单（上下文太大），任务不够有挑战性
- **Random 问题：** 随机撒点遮罩，预测时上下文分布太散，不聚焦

**Multi-block 的好处：** 多个目标块让预测任务有足够挑战，同时上下文块足够大（85%~100%）提供了丰富信息。

## 实验结果

### ImageNet 线性探测（特征质量的核心指标）

模型训完后**冻结权重**，只在上面训练一个线性分类器——这个指标反映的是"模型自己学到的特征有多好"。

| 方法 | 架构 | 是否需要数据增强 | Top-1 准确率 |
|------|------|:---:|:---:|
| **I-JEPA** | ViT-H/14 (300 epoch) | ❌ 不需要 | **79.3%** |
| I-JEPA | ViT-H/16@448 (300 epoch) | ❌ 不需要 | **81.1%** |
| MAE | ViT-H/14 (1600 epoch) | ❌ 不需要 | 77.2% |
| data2vec | ViT-L/16 (1600 epoch) | ❌ 不需要 | 77.3% |
| CAE | ViT-L/16 (1600 epoch) | ❌ 不需要 | 78.1% |
| DINO | ViT-B/8 (300 epoch) | ✅ 需要 | 80.1% |
| iBOT | ViT-L/16 (250 epoch) | ✅ 需要 | 81.0% |

**关键发现：**

1. I-JEPA **碾压**了其他不用数据增强的方法（MAE、data2vec、CAE）
2. I-JEPA **只用了 300 epoch 就超过了 MAE 的 1600 epoch**
3. I-JEPA ViT-H/14 的算力消耗**比 iBOT ViT-S/16 还少**（iBOT 的 smallest 模型比 I-JEPA 的 largest 模型还费算力！）
4. 把分辨率抬到 448，I-JEPA 能追上甚至超过用数据增强的 DINO

### 半监督学习（1% 标签）

只给 1% 的 ImageNet 标签（每类 12-13 张图），看模型能从中学到什么程度：

| 方法 | Top-1 |
|------|:---:|
| **I-JEPA ViT-H/16@448** | **77.3%** |
| I-JEPA ViT-H/14 | 73.3% |
| MAE ViT-H/14 | 71.5% |
| data2vec ViT-L/16 | 73.3% |
| iBOT ViT-B/16 | 69.7% |
| DINO ViT-B/8 | 70.0% |

I-JEPA 在极少量标签下表现最好。

### 低级视觉任务（物体计数、深度预测）

这是 I-JEPA 最惊艳的地方——**不变性方法（DINO、iBOT）在这个任务上表现很差**，因为它们学的是"不变的特征"（不管怎么变都编码成一样），损失了位置、数量、深度这类空间信息。

但 I-JEPA 需要**预测"某个位置有什么"**，天然保留了空间信息：

| 方法 | 物体计数 (Clevr/Count) | 深度预测 (Clevr/Dist) |
|------|:---:|:---:|
| **I-JEPA ViT-H/14** | **86.7** | **72.4** |
| MAE ViT-H/14 | 90.5 | 72.4 |
| data2vec ViT-L/16 | 85.3 | 71.3 |
| DINO ViT-B/8 | 86.6 | 53.4 |
| iBOT ViT-L/16 | 85.7 | 62.8 |

**深度预测方面，I-JEPA 大幅领先于 DINO（+19%）和 iBOT（+9.6%）。**

### 算力效率

| 方法 | 架构 | GPU 小时 | ImageNet-1% 精度 |
|------|------|:---:|:---:|
| **I-JEPA** | ViT-H/14 | **~1100** | **73.2%** |
| MAE | ViT-H/14 | ~6000 | 71.5% |
| data2vec | ViT-L/16 | ~8000 | 73.3% |
| iBOT | ViT-S/16 | ~2200 | 66.0% |

I-JEPA **只用不到 MAE 1/5 的算力**就打平甚至更好。

## 为什么 I-JEPA 效果这么好？（论文没明说，但可以从实验推导）

### 原因 1：抽象空间自动过滤了无关细节

MAE 要预测被遮住区域的**每一个像素**——包括树皮的纹理、背景的噪点、光照的渐变。模型必须花大量容量去学这些"生成细节"。

I-JEPA 不需要。目标编码器在抽象空间中已经自动过滤了这些无关细节，留下的就是**语义信息**——"这里是一个猫头"而不是"这个像素的 RGB 值是 (128, 95, 73)"。

```
MAE 的预测目标：  [128, 95, 73, 131, 92, 70, ...]  ← 每个像素都重要
I-JEPA 的预测目标：[0.32, -0.47, 0.81, ...]         ← 抽象语义
```

### 原因 2：EMA 目标编码器提供了稳定的学习目标

目标编码器的权重是上下文编码器的**慢速移动平均**，这有两层好处：

1. **目标稳定** — 每次迭代目标变化很小，训练信号平滑
2. **防止塌缩** — 如果上下文编码器试图把所有图片编码成同一个值，目标编码器跟不上，损失会爆炸

### 原因 3：目标块大小在"语义尺度"上

目标块占图片的 **15%~20%**，这个尺寸恰好对应"一个有意义的物体部件"——猫头、车轮、窗户角、人的脸。如果目标块太小（<5%），它就变成了"像素纹理预测"；如果太大（>30%），它太容易猜，学不到好东西。

论文的消融实验清清楚楚地展示了这一点：

| 目标块大小 | 1% ImageNet 精度 |
|:---:|:---:|
| (0.05, 0.1) | 31.2% |
| (0.15, 0.2) | **54.2%** |
| (0.3, 0.4) | 47.1% |

## 预测器的可视化（Figure 6 的故事）

论文做了一个非常漂亮的可视化实验：

1. 用 I-JEPA 训练好编码器和预测器
2. 冻结编码器和预测器
3. 额外训练一个**生成式解码器**（RCDM 框架），把预测器的输出**解码回像素**

结果（见论文 Figure 6）：

```
原始图片:         [🐦 站在树枝上]
上下文图片:       [背景模糊，鸟的轮廓可见]
预测器解码样本 1: [🐦 清晰的鸟身 + 模糊的鸟头姿势 A]
预测器解码样本 2: [🐦 清晰的鸟身 + 模糊的鸟头姿势 B]
预测器解码样本 3: [🐦 清晰的鸟身 + 模糊的鸟头姿势 C]
```

**观察：** 预测器知道"鸟身"的精确位置（多个样本中一致），但对"鸟头"的精确位置有不确定性（多个样本中不同姿势）。

这说明了 I-JEPA 预测器的本质：

- **捕获了确定性信息**：鸟在图片中的位置、姿态方向
- **丢弃了不确定性信息**：鸟头的精确角度、羽毛的纹理细节、背景的精确像素

**这就是"在抽象空间里做预测"的意思——只留下重要的，丢掉不重要的。**

## I-JEPA vs LeCun 2022 蓝图

| LeCun 2022 蓝图中的模块 | I-JEPA 实现了多少 |
|------|:---:|
| **感知模块（Perception）** | ✅ 上下文编码器 + 目标编码器 |
| **世界模型（World Model）** | ✅ 预测器（在一个层次上做抽象预测） |
| **能量/代价模块（Cost）** | ❌ 未实现（简化成 L2 损失） |
| **动作生成器（Actor）** | ❌ 未实现（I-JEPA 是纯视觉编码器） |
| **配置器（Configurator）** | ❌ 未实现 |
| **分层（Hierarchical）** | ❌ 未实现（只有单层预测） |

I-JEPA 实现了蓝图中最核心的部分——**在抽象空间里做预测**——但去掉了蓝图中的复杂组件（能量函数、动作生成、配置器、分层结构）。这恰恰是它聪明的地方：**只把最核心的思想拿出来验证，而不是一口气做完整个系统。**

## 局限性与后续工作

**I-JEPA 的局限性：**

1. **只有一层** — LeCun 蓝图是 H-JEPA（层次化），I-JEPA 只有一个预测层
2. **只做图像** — 没有像 data2vec 那样同时覆盖语音、文本
3. **依赖 ViT** — I-JEPA 的架构设计基于 Vision Transformer，在 CNN 上不一定有效
4. **线性探测优势 ≠ 微调优势** — I-JEPA 在线性探测（冻结编码器）上表现极好，但端到端微调（fine-tuning）下未必比 MAE 强

**后续工作：**

- **V-JEPA**（Video JEP-A，Meta 2024）— 把 JEPA 扩展到视频，只处理 1/4 的帧就能学到好特征
- **data2vec-v2** — 类似思路，但覆盖了视觉、语音、文本
- **ImageWorld** — 在 I-JEPA 基础上加入了更大的数据集和更多层次的预测

## 一句话带走

> **别再让模型去猜每个像素的颜色了。让它猜"这块区域在抽象空间里应该编码成什么向量"——又省算力、又学得深、还能保留空间位置信息。这就是 I-JEPA 做的事。**

I-JEPA 是 JEPA 理念的**第一次成功落地实验**。它不花哨、不复杂（损失函数就是简单的 L2 距离），但实验做得极其扎实，消融实验层层剥开，把"为什么在抽象空间预测比在像素空间好"这件事证明得清清楚楚。

## 核心公式

### 1. I-JEPA 损失函数（论文核心公式）

```
L = (1/M) · Σᵢ₌₁ᴹ  D( ŝ_y(i), s_y(i) )

其中：
  D(ŝ_y(i), s_y(i)) = Σ_{j ∈ B_i}  ||ŝ_yⱼ - s_yⱼ||²
```

| 符号 | 含义 |
|------|------|
| M | 目标块数量（默认 4） |
| B_i | 第 i 个目标块包含的补丁集合 |
| ŝ_yⱼ | 预测器输出的第 j 个补丁的表示 |
| s_yⱼ | 目标编码器输出的第 j 个补丁的表示 |

白话：**把 M 个目标块的所有补丁的"猜测值"和"真值"的平方差加起来取平均。猜得越准，损失越低。**

### 2. 目标编码器权重更新（EMA）

```
θ̄_t = m · θ̄_{t-1} + (1 - m) · θ_t

其中：
  m = 0.996 → 1.0（线性增加）
```

| 符号 | 含义 |
|------|------|
| θ̄ | 目标编码器权重 |
| θ | 上下文编码器权重（通过梯度更新） |
| m | 动量系数（训练开始时 0.996，逐渐增加到 1.0） |

白话：**目标编码器是上下文编码器的"慢动作版本"，每次只跟着上下文编码器变动 0.4% 的比例。这样学习目标稳定，不会塌缩。**

### 3. I-JEPA 的前向过程

```
上下文块 x → 上下文编码器 f_θ → sₓ = f_θ(x)

预测器输入 = sₓ + 位置编码(mask_tokens)

ŝ_y = g_Φ( 预测器输入 )         ← 预测的目标表示

完整图片 y → 目标编码器 f_θ̄ → s_y = f_θ̄(y)  ← 真实的表示（目标）

损失: L = ||ŝ_y - s_y||²         ← 在表示空间里算 L2 距离
```

### 4. 与 MAE（像素重建）的损失对比

```
MAE 损失（像素空间）:
  L_MAE = ||D(z_masked) - y_pixels||²
          ↑ 解码器重建的像素          ↑ 原始像素

I-JEPA 损失（表示空间）:
  L_I-JEPA = ||g(f_θ(x)) - f_θ̄(y)||²
              ↑ 预测器输出的表示   ↑ 目标编码器输出的表示
```

**关键区别：** MAE 需要解码器把隐向量"展开"回像素空间，解码器消耗大量参数。I-JEPA 不需要解码器——直接在表示空间比大小。

### 5. Masking 策略的参数

```
目标块:
  尺寸范围: scale ∈ (0.15, 0.2)    ← 占图片面积的 15%~20%
  宽高比: aspect_ratio ∈ (0.75, 1.5)
  数量: M = 4                          ← 每次预测 4 个目标块

上下文块:
  尺寸范围: scale ∈ (0.85, 1.0)    ← 占图片面积的 85%~100%
  宽高比: 1.0 (正方形)
  后处理: 去掉与目标块重叠的区域
```

### 公式逻辑关系图

```
训练流程（一条线串起来）:

输入图片 y
    │
    ├──→ 目标编码器 f_θ̄ → s_y = f_θ̄(y)    ← 真实目标
    │
    └──→ 采样上下文块 x + 目标块 {B_i}
              │
              ▼
         上下文编码器 f_θ → sₓ = f_θ(x)
              │
              ▼
         预测器 g_Φ → ŝ_y(i) = g(sₓ, pos_B_i)  ← 预测目标
              │
              ▼
         损失: L = (1/M) Σᵢ Σ_{j∈B_i} ||ŝ_yⱼ - s_yⱼ||²
              │
              ▼
         梯度更新 f_θ (AdamW)
              │
              ▼
         EMA 更新 f_θ̄ = m·θ̄ + (1-m)·θ
              │
              ▼
         下一张图片，重复
```

**最终结果：** 上下文编码器 f_θ 学会了从部分观察中提取"有用的抽象信息"，而目标编码器 f_θ̄ 学会了将完整图片编码成"稳定的语义表示"。训练结束后，目标编码器就是你要的那个**强大的视觉特征提取器**。
