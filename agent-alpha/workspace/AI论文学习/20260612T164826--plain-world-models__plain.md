# World Models — 白话版

> **来源：** World Models, David Ha & Jürgen Schmidhuber, 2018 (arXiv)
>
> **作者：** David Ha, Jürgen Schmidhuber
>
> **阅读日期：** 2026-06-12

---

想象你第一次玩赛车游戏。屏幕上是赛道画面，你按上下左右控制车子。你撞了几次墙才知道"这个弯要提前减速"——你在脑子里建了一个模糊的图：方向盘往哪打，车会往哪走。下次再遇到差不多的弯，你不用真的撞一次墙也能提前准备。

这就是"世界模型"的意思。这篇论文的核心就一句话：**让 AI 在脑子里建一个"模拟器"，然后在这个模拟器里练习，不用每次都去真实世界里撞墙。**

作者是 David Ha 和 Jürgen Schmidhuber。Schmidhuber 是那种在 AI 界待了三十年、很多重要想法其实最早都是他提的大前辈。2018 年这篇论文发出来的时候，大家正在拼命让 AI "学更多的东西、用更少的真实数据"——这个方向到今天也是热的。

他们用一个 2D 赛车游戏做实验。一辆小车在椭圆赛道上跑，AI 只能看到屏幕像素——就是一张张小图片。它要学的是"怎么一直跑下去不掉出赛道"。

### 问题在哪？

大多数做法是让 AI 直接在游戏里试错——撞出去、记住、下次别这样。这叫"强化学习"，听着高级，说白了就是"反复试、挨了打就长记性"。但这事儿在真实世界里很贵。你要让一个机器人学走路，它摔倒一次就是一次硬件磨损。你要让一辆自动驾驶车在真实路面上学，撞一次就是真的撞。

那能不能让 AI 先在脑子里"想象"一下？就像你闭上眼睛在脑子里过一遍"这个弯我怎么过"，不用真上车。

这就是这篇论文的突破口。

### 他们怎么做？

他们把问题拆成三块，各配一个"小脑"：

**第一块叫"眼睛"。** 赛车游戏的画面很复杂——树啊云啊赛道啊，几百个像素点。但真正有用的信息可能就几个：车在赛道的哪个位置、车头朝哪、速度多快。"眼睛"的工作就是把一张大图片压缩成一小串数字——几十个就够了。这些数字叫"隐变量"（latent variables），说白了就是"画面里真正重要的那点东西"。

**第二块叫"预测器"。** 它拿"眼睛"压缩出来的那串数字，再加上 AI 打算做什么动作（左转、加速、不减速），然后预测下一秒会发生什么——"如果你现在左转，下一帧的画面会变成这样"。这个预测器有一个特别的地方：它不是铁口直断"下一秒一定是 X"，而是说"下一秒可能是 X，可能性 Y%"——它知道世界有随机性，不是所有事都能算死。这个设计叫"混合密度网络"（MDN），是他们特意加进来的，因为真实世界的变化本来就不是一条线。

**第三块叫"舵手"。** 它决定"现在该做什么"——左转还是右转、油门踩多深。它读的是"眼睛"给的当前状态和"预测器"说的未来可能性，然后拍一个动作出来。

```
┌─────────────────────────────────────────────────────┐
│                  世界模型 (World Model)              │
│                                                       │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐      │
│   │  眼睛    │    │ 预测器   │    │  舵手    │      │
│   │ (VAE)    │───→│(MDN-RNN) │←───│(Controller)│    │
│   │ 看画面   │    │ 猜未来   │    │ 做决定   │      │
│   └──────────┘    └──────────┘    └──────────┘      │
│        ↑                                ↓            │
│        │          ┌──────────┐          │            │
│        └──────────│ 真实世界 │──────────┘            │
│                   │ (赛车游戏)│                       │
│                   └──────────┘                       │
└─────────────────────────────────────────────────────┘
```

### 神来之笔在哪？

常规思路是：眼睛看画面 → 舵手决定动作 → 动作在游戏里跑 → 看结果好不好 → 调整舵手。

但他们的做法是：先把"眼睛"和"预测器"在真实游戏里训练好——让眼睛学会看画面、让预测器学会猜下一帧——然后，**把整个"模拟器"从游戏里抽出来，搬到 AI 的脑子里。接着，让"舵手"完全在这个脑子里练习，不再需要真实游戏了。**

舵手在脑子里练习的方式很直接：它想象一个初始画面，然后对自己说"如果我往左打，预测器说接下来会是这样……那我再往右拉一点会怎样……"——它就在这个想象世界里跑了一遍又一遍，像你在脑子里模拟一道面试题的各种答法。

练好了，再放回真实游戏里。它已经"见过"各种情况了，不会像新手一样一出弯就懵。

```
   训练阶段 1                   训练阶段 2
┌──────────────┐           ┌──────────────┐
│  真实游戏     │           │   脑内模拟器  │
│              │           │              │
│ 眼睛 ← 学看  │           │ 舵手 ← 练习   │
│ 预测器 ← 学猜 │           │ (完全在梦里)  │
│ 舵手 ← 先不管 │           │              │
└──────────────┘           └──────────────┘
                                   │
                                   ▼ 练好了放回去
                            ┌──────────────┐
                            │  真实游戏     │
                            │  直接跑完整圈  │
                            └──────────────┘
```

### 结果呢？

效果出奇地好。小车在模拟器里练了几百世代就能在真实赛道上稳稳跑完整圈。而且他们做了一个特别有意思的测试：如果只给"舵手"（控制信号的噪音）而不让它看到具体的视觉画面，它也能跑——它学会了"就按这个节奏加油和转向"的肌肉记忆。

最反直觉的发现是这个：**用世界模型生成的"梦境数据"来训练舵手，比用真实游戏数据训练效率高得多。** 因为模型可以自己在脑子里"造"出各种边缘情况——那些在真实游戏里可能要跑几十圈才能遇到一次的罕见弯道。

### 有什么代价？

世界模型的质量决定了它想象的世界有多准。如果预测器学歪了，它在脑子里跑得再好，回到真实世界也是错的——这叫**"模型偏差"**（model bias）。就像你想象中自己投篮百发百中，真站到球场上一投发现根本不是那么回事。这篇论文用了一个相对简单的赛道环境来避开这个问题，但真实世界的复杂度要远高于此。

另外，这套结构里"眼睛"（VAE）和"预测器"（MDN-RNN）最初还是要在真实环境里训练的——你没完全摆脱真实数据，只是把最贵的那部分（试错训练）搬到了脑内。

### 这颗内核是什么？

> **聪明不是从零开始试出来的。聪明是先在脑子里建一个世界的简化模型，然后在这个模型里推演。你不需要真的把每种死法都死一遍才知道怎么活。**

这篇论文的 DNA 后来长进了很多地方：Dreamer（从世界模型里学规划）、DayDreamer（在真实机器人上跑世界模型）、还有各种基于模型的强化学习方法。今天很多游戏 AI、机器人控制的研究背后，都站着一个能在脑子里"先想一步"的模型。

### 核心公式（附白话翻译）

论文里的数学其实不多，三套公式对应三个组件。下面是每个公式的白话版翻译。

#### ① 眼睛（VAE）→ 压缩画面的公式

VAE（变分自编码器）做两件事：
- **编码器** `Encoder(x_t)`：输入画面 x_t，输出一串数字 z_t（均值 μ 和方差 σ²），核心是 **"把 4096 个像素压缩成 32 个关键数字"**
- **解码器** `Decoder(z_t)`：拿这 32 个数字，**"反向还原成一张画面"**

VAE 学习靠两条绳拽着：

```
损失 = 重建误差 + 正则项
     = "还原的画面跟原图差多少" + "压缩后的数字分布别太怪"
```

公式写法：

`L_VAE = -E[log P(x|z)] + KL(q(z|x) || p(z))`

白话翻译：
- 第一项 `-E[log P(x|z)]` — **还原要多像？** 解码器把 z 还原成画面，跟原图比，差距越小越好
- 第二项 `KL(q(z|x) || p(z))` — **压缩要有多规矩？** 编码器输出的数字分布别飘太远，尽量靠近标准正态（均值为 0、方差为 1 的钟形曲线）

**直观理解：** VAE 在找一个"信息最浓缩又不失真"的表达方式——既要能完整还原画面（像 ZIP 压缩），又要让压缩后的数字规矩好使（像普通话而不是方言）。

#### ② 预测器（MDN-RNN）→ 猜下一秒的公式

循环神经网络（RNN）维护一个"记忆状态" h_t，每走一步都更新：

`h_t = RNN(h_{t-1}, z_{t-1}, a_{t-1})`

白话翻译：**"记住上一步的记忆 h_{t-1}、上一帧的画面压缩 z_{t-1}、上一步的动作 a_{t-1}，三步合一，算出此刻的新记忆 h_t"**

有了记忆 h_t，MDN 就猜"下一帧画面压缩值 z_t 大概长什么样"。它不说唯一答案，而是说一个**混合猜测**——给出好几个可能，每个带权重（好比"70% 的可能你加速后会冲出去，30% 的可能刚好过弯"）：

`P(z_t | h_t) = Σ π_i · N(z_t | μ_i, σ_i²)`

白话翻译：
- `N(z_t | μ_i, σ_i²)` — **第 i 种猜测**：下一帧压缩值大概在 μ_i 附近，误差 σ_i
- `π_i` — **这个猜测有多靠谱**（所有 π_i 加起来 = 1）
- 整个公式 = **"把好几个钟形曲线按权重叠起来，得到一个多峰的猜测分布"**

论文用 5 个高斯分布（i = 1 到 5）来叠，足以覆盖大多数赛道变化。

损失函数：

`L_MDN = -log P(z_t | h_t)`

白话翻译：**"真实发生的那一帧，模型猜对了的概率有多大？取负对数——猜得越准，损失越小。"**

#### ③ 舵手（Controller）→ 做决定的公式

舵手是最简单的：一个神经网络把"当前画面压缩 + 当前记忆"映射成一个动作：

`a_t = Controller(z_t, h_t)`

白话翻译：**"看一眼当前画面 z_t，回想一下之前的事 h_t，决定下一步是左转还是右转、油门踩多少。"**

舵手怎么训练？不用梯度下降，用**进化策略**（CMA-ES）：

```
目标：最大化 J(θ) = E[ Σ reward_t ]
白话：反复跑 → 记下谁的跑法得分高 → 把跑得差的随机调一调 → 
      保留跑得好的参数 → 在好参数附近再随机试 → 循环
```

这和达尔文进化论一模一样——"把优秀基因的后代稍微变异一下，一代代自然筛选"。

#### ④ 做梦（Dreaming）→ 整个系统运作公式

一旦 VAE 和 MDN-RNN 训练好，"做梦"的流程就是：

```
1. 从真实世界拿第一帧 x₀ → z₀ = Encoder(x₀)   // 看一眼
2. 初始化 h₀ = 0                                // 清空记忆
3. 循环每一步：
   a. a_t = Controller(z_t, h_t)                // 舵手做决定
   b. z_{t+1} ~ MDN-RNN(z_t, a_t, h_t)          // 预测器脑补下一帧
   c. h_{t+1} = RNN(h_t, z_t, a_t)              // 更新记忆
4. 用 z_t 序列算每一帧的奖励，总和 = 这一圈跑得怎样
```

舵手只能在梦里训练时，奖励怎么算？论文假设"在压缩空间 z 里就能判断跑得好不好"——这依赖一个额外的**奖励预测器**（从 z 和 h 预测当前帧的即时奖励）。

---

### 一句话带走

**先让 AI 学会"猜这个世界怎么动"，再让它关起门来练——你会发现它醒着的时候跑得比醒着练的还好。**
