Skill 安装机制对比

OpenClaw、Hermes、agent-alpha 在“安装、存储、配置、更新、调用、删除 skill”上的机制差异。重点场景是：用户给出 Panniantong/Agent-Reach.git，并希望自动处理 npm / pip 依赖。

OpenClaw

把 skill 包管理和依赖安装拆开。ClawHub 更像 skill 包管理器；OpenClaw 核心读取 skill frontmatter 里的结构化 install spec，并能执行 brew、node、go、uv、download。

结构化依赖安装 bin 缺失检查 多目录加载

Hermes

有完整 hub：GitHub、URL、tap、索引等来源，安装前 quarantine 和安全扫描，安装后写 lock，支持 update / uninstall / enable / disable。

Hub 生命周期 Lock + Hash 多来源安装

agent-alpha

当前主要是本地加载器：扫描 skills/SKILL.md，注册 load_skill。能在 bash 权限层识别 npm / pip 命令，但没有 skill 安装生命周期。

本地扫描 load_skill bash 审批

生命周期视图

1. 来源仓库、registry、URL、内置目录、本地目录。
2. 安装下载或复制 skill 文件，确定安装目录。
3. 依赖检查 bin、解析 install spec、执行 npm / uv / go 等。
4. 管理记录 lock、hash、来源、启用禁用、更新删除。
5. 调用slash command、skill_view、load_skill 或 tool dispatch。

依赖安装差异

OpenClaw 把依赖写入 skill metadata，再由 installer 执行。它知道要检查哪些 bin，也知道用 npm、uv、go、brew 或 download 中的哪一种安装。

这使得“一个 skill 需要某个 CLI 工具”变成可检查、可安装、可报错的流程。

包管理差异

Hermes 更像 skill hub。它能记录来源、文件、hash、安装路径、扫描结果，并通过 lock 做更新和卸载。

这使得“一个 skill 从哪里来、现在是不是最新、能不能删除”变成可管理状态。

agent-alpha 当前边界

agent-alpha 当前有 SkillLoader、ToolLoader 和 load_skill，所以它能使用本地已有 skill。它也能在 command_path_extractor.py 和 sandbox_guard.py 里识别 npm / pip 安装命令需要审批。但它还没有把“安装 skill”作为一个独立生命周期来处理。
