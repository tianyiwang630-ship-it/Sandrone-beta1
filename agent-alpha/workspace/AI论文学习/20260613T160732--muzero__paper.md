# 不用规则，也能规划

**Mastering Atari, Go, Chess and Shogi by Planning with a Learned Model**

Schrittwieser et al., 2020 · Nature 588, 604–609

---

## 问题

### 主角

一个想学会在未知环境里做规划的智能体。它面前不是棋盘，是它从未见过的游戏——它不知道摁下哪个键会让球弹回来，不知道落子在哪儿才算"围住"，不知道马的走法，不知道对方能不能一步将死它。它只能靠试、靠看、靠自己脑子里长出那个世界的模型。

### 困境

规划的核心是一个简单直觉：你在走之前先"想一遍"。下棋的人想三步，打游戏的人在脑子里模拟"如果我往左跳会怎样"。这个直觉在计算机里叫**树搜索**——从当前状态展开所有可能动作，评估每个动作的后果，选最好的那条路。

问题是：要"想"，你得先知道这个世界怎么运转。

AlphaGo、AlphaGo Zero、AlphaZero 这一路，靠的都是**提前把规则告诉算法**。在围棋里，算法知道"落子后棋盘变什么样"；在象棋里，算法知道"马走日象走田"；在 Atari 上，它却不知道——因为 Atari 的规则写在 ROM 里，是闭源的、像素级的、没有显式状态转移函数的。所以 AlphaZero 那一套到了 Atari 上就卡住了：它需要一个完美的模拟器（simulator），而现实世界没有模拟器给你。

更致命的是——**现实世界几乎从来不给规则说明书**。自动驾驶不知道前车下一秒会不会刹车，机器人不知道门把手转多少度会开，医疗决策系统不知道病人的病情下一步会往哪个方向走。如果规划算法只能在"规则已知"的世界里工作，它就永远困在棋盘上。

### 旧路

在此之前，有三条路，每条都有硬伤：

**第一条：基于模型的强化学习（Model-based RL）**——让智能体先学一个环境模型（比如"这个按键会让画面变成怎样"），再用这个模型做规划。听起来就是 MuZero 要做的事。但问题是，学出来的模型误差会累积：预测第一步差 1%，预测到第十步差到离谱，规划全错。更糟的是，传统做法要求学出来模型**重构全部观测**——你要精确预测下一帧画面的每一个像素——这在高维空间（Atari 的 160×210 像素画面）里几乎不可能，而且视频游戏里一个像素的变化就能让整个规划脱轨。

**第二条：无模型强化学习（Model-free RL）**——不学模型，直接学"看到这个画面，该按哪个键"。DQN、Rainbow 都是这条路。它们不需要模拟器，直接用真实交互学策略。但它们的代价是**样本效率极差**——玩几千万帧才能学会一个游戏，而且不能泛化到没见过的状态。因为它们每一步都是"盲走"，不在脑子里过一遍"如果……会怎样"。

**第三条：AlphaZero 范式**——用完美的先验规则做蒙特卡洛树搜索（MCTS），在搜索中规划，再反过来提升策略。这是当时最强的规划算法，在围棋、象棋、将棋上都达到超人水平。但它**依赖规则的硬编码**。没有规则，它的一整套机制就塌了。

所以困境很清楚：**要么你会规划但需要别人告诉你规则（AlphaZero），要么你不需要规则但不会规划（DQN）。能不能两个都要？**

## 翻译

### 转折：作者看到了什么别人没看到的东西？

作者（Julian Schrittwieser 和他的 DeepMind 团队）看到了一个关键判断：**做规划不需要精确的世界模型，只需要一个"够用"的世界模型。**

什么叫"够用"？不是预测下一帧画面长什么样，不是重构全部视觉细节，而是一个更窄、更精准的目标——**预测三样东西就够了**：

1. 做了这个动作后，状态会变成怎样（但不是在像素上变，而是在一个隐空间里变）
2. 做了这个动作后，能拿到多少奖励（这一步得分多少）
3. 这个新状态是不是"好"（值函数——给定当前状态，未来还能拿到多少奖励）

这个判断把整个问题拧过来了：**不要学一个能生成画面的世界模型，要学一个能用来做规划的世界模型。** 两者需要的精度完全不同——前者要像素级，后者只需要在"哪步好哪步差"这个维度上准。

### 解法：MuZero 的三块积木

MuZero 由三个神经网络组成，它们在训练时一起学，在规划时一起用：

**第一块：表示函数（Representation Function）——把画面压缩成隐状态**

智能体看到的原始画面是 210×160 像素的彩色图像。表示函数 `h` 把这一大堆像素压缩成一个紧凑的隐状态（hidden state）向量 `s^0`。这个隐状态不是像素，也不是人能看懂的"棋盘局面"，它是为规划专门压缩出来的——只保留做决策需要的信息。

```
原始画面 (210×160)  →  h  →  隐状态 s⁰ (一个小向量)
```

**第二块：动力学函数（Dynamics Function）——在脑子里推演**

这是 MuZero 的核心创新。给定当前隐状态 `s^k` 和一个候选动作 `a^{k+1}`，动力学函数 `g` 预测两个东西：
- 下一个隐状态 `s^{k+1}`（在隐空间里，不在像素上）
- 这一步的即时奖励 `r^{k+1}`（比如吃了一个子得 1 分）

关键就在这里：**这个预测完全发生在隐空间里。** MuZero 不需要把 s^{k+1} 解码回像素，就能继续往下推演——它整个规划过程都在脑子里走，不把"想象画面"倒出来。

```
隐状态 s^k + 动作 a^{k+1}  →  g  →  新隐状态 s^{k+1} + 奖励 r^{k+1}
```

**第三块：预测函数（Prediction Function）——评估局面好坏**

给定一个隐状态 `s^k`，预测函数 `f` 输出两个东西：
- 策略头（policy head）：在当前局面下，每个动作应该被选中的概率
- 值头（value head）：当前局面的预期回报（如果从这个局面接着玩，能赢的概率多大/总分多高）

```
隐状态 s^k  →  f  →  策略 π^k + 值 v^k
```

这三块合起来，MuZero 就能在脑子里做蒙特卡洛树搜索（MCTS）了——即使它从来没被告知游戏规则。

### 规划过程：在脑子里下一盘棋

假设 MuZero 在玩围棋。它看到一个棋盘画面，但不知道"围住"是什么意思。它怎么思考？

1. **表示阶段**：画面通过 `h` 压缩成隐状态 s⁰（这是根节点）
2. **展开阶段**：对 s⁰，预测函数 f 给出初步策略 π⁰ 和值 v⁰（"第一手可能落在这里的概率高"）
3. **搜索阶段**：MCTS 做多次模拟——
   - 根据当前策略选择一个动作 a¹（不是瞎选，是照着当前"猜测"选最有前途的）
   - 动力学函数 g 推演 a¹ 的结果：新隐状态 s¹ + 即时奖励 r¹
   - 在 s¹ 上，预测函数 f 给出策略 π¹ 和值 v¹
   - 继续从 s¹ 展开：选动作 a² → g 推演到 s² → f 评估 s²……
4. **反向传播**：每模拟完一条路径，搜索结果（值 + 奖励）反向传播回路径上的所有节点，更新它们的统计信息（访问次数、平均价值）
5. **选最终动作**：搜索结束后，根据根节点的访问次数分布选动作——被搜索访问最多的动作就是最终决策

整个过程里，MuZero **一次都没用到真正的游戏规则**。它不知道围棋的"气"是什么，不知道"提子"怎么发生的。它只是在一个学出来的隐空间里不断推演——"如果我往这里放子，这个隐状态会变成那样，对应那个奖励和那个值"——然后在真实环境中执行搜索找到的最优动作。

MuZero 学到的东西，不是"围棋规则"，而是一个**把动作序列映射到胜负的隐式模型**。这个模型在隐空间里的推演精度恰好够规划用，但不需要精确到像素或合法落子点——它只需要知道"这个选择比那个选择好"。

### 训练过程：在想象中学习，在学习中想象

三块网络是联合训练的，训练的损失函数（loss）有三个来源：

1. **奖励损失**：预测的即时奖励 r^k 和真实奖励 r* 之间的差距（交叉熵）
2. **值损失**：预测的值 v^k 和通过实际游戏结果计算出的目标值之间的差距（均方误差或交叉熵）
3. **策略损失**：预测的策略 π^k 和搜索后改进的策略 π* 之间的差距（交叉熵）

每个训练样本来自实际游戏数据（存储在 replay buffer 中）。MuZero 从 replay buffer 里采样一个游戏片段，用当前的表示函数编码起始状态，然后用动力学函数在隐空间里展开 K 步（通常是 5 步），每一步都计算上述三个损失，然后反向传播更新所有三个网络的参数。

这才是 MuZero 最漂亮的地方：**它自己在脑子里推演 K 步，然后拿这 K 步的预测结果和游戏真实记录对比，来学习怎么在脑子里推演得更准。** 它不是从外部学规则，而是从自己的搜索经验中学"怎样想才想得对"。

这和人类学习下棋的过程很相似：你刚开始不知道"双车错"是什么，但你在脑子里摆了几百个变化之后，自然就知道了"把两个车放在同一列上很危险"——你不是背诵了规则，是**推演多了自然学会的**。

### 在 Atari 上的独特挑战

Go、Chess、Shogi 是确定性的、离散动作的、奖励稀疏的。Atari 不一样：

- 画面是像素级的，高维而充满视觉噪音
- 动作不是离散的"落子点"，是几个按钮（上、下、左、右、A、B）
- 奖励信号频繁（每帧都有分数变化）
- 部分游戏有随机性（比如打砖块里球的轨迹）

MuZero 能在 Atari 上工作，靠的是它**不做像素级预测**——如果它要预测下一帧画面长什么样，在 Atari 上会疯掉。但它只预测隐状态和奖励，所以视觉复杂性不影响规划精度。同时，它使用一个 15 帧的 stacking（堆叠帧缓冲）来捕获运动信息，用缩放后的画面（96×96 灰度）降低输入维度。

结果是：在 57 个 Atari 游戏上，MuZero 的平均表现超过了此前最强的无模型算法（DQN、Rainbow、R2D2），也超过了使用完美模拟器的 AlphaZero 变体——**它不需要规则，反而超过知道规则的人。**

## 核心概念

### 隐空间中的规划

传统规划算法在"状态空间"里做搜索——状态是棋盘格、是像素、是明确定义的位置坐标。MuZero 的规划在"隐空间"里做——状态不是人能理解的，但它恰好保留了做决策需要的信息。

这个选择解释了为什么 MuZero 能同时搞定围棋（低维但极端复杂的状态空间）和 Atari（高维但相对简单的动作空间）。隐状态是一个**自动学习的压缩表示**——它丢掉了一切与决策无关的视觉细节，保留了一切与胜负相关的结构信息。

你可以把它理解为：MuZero 的"脑海"里没有棋盘图像，没有棋子，没有像素。它有一个**抽象的局面感**——"目前这个形势（隐状态）下，走这一步（动作），形势会变成那个样子（新隐状态），大概能得这么多分（奖励），最终走势大概是这样（值）。"

这种抽象就是规划的关键：**精确的抽象比精确的重构更有用。**

### 从 AlphaZero 到 MuZero 的关键一步

AlphaZero 的流程是：

```
真实状态 + 规则 → 搜索 → 策略 & 值
```

MuZero 的流程是：

```
真实状态 → 表示函数 → 隐状态 → 动力学函数(隐空间推演) + 预测函数 → 搜索 → 策略 & 值
```

AlphaZero 需要规则来回答"如果我走这一步，局面变成什么样"。MuZero 用学出来的动力学函数回答了这个"如果……会怎样"——在隐空间里。

这个迁移本质上是一种**抽象层次的转移**：AlphaZero 在"具体局面"层做推演，MuZero 在"抽象表征"层做推演。后者的抽象程度更高，所以能跨领域使用同一套机制。

### 自我对弈 + 规划 = 数据的永动机

MuZero 不需要人类数据，不需要规则手册。它只需要一个能和真实环境交互的接口（拿了动作返回画面和奖励）。

- 它用当前的策略和搜索来玩（生成数据）
- 把玩过的游戏存入 replay buffer
- 从 buffer 里采样，用当前模型做 K 步隐空间推演
- 计算推演和真实之间的损失，更新网络
- 更新的网络玩得更好，生成更好的数据
- 循环

这就是强化学习的"永动机"——只要有环境交互，数据就在自己长。唯一的限制是算力。

## 洞见——这个故事的真正内核

这篇论文讲的根本不是一个算法。它讲的是一句话：

### **真正的智能不是"知道规则"，而是"能在未知中发现规则的结构"**

AlphaZero 证明了：如果你告诉它规则，它就能成为世界上最好的棋手。MuZero 证明了更硬的一件事：**即使你不告诉它规则，它自己也能从交互中学会"足够用来规划的世界模型"**——而且这个学出来的模型，好到不需要真实规则。

这个结论对人类学习的隐喻是深刻的。人类的大多数学习，也不是靠先读说明书再动手的。你学骑车的时候没人给你动力学方程，学做饭的时候没人给你化学反应式，学谈判的时候没人给你博弈矩阵。你是在试、在摔、在观察、在**脑子里重放和推演**——而这正是 MuZero 做的事。

更深一层的含义：**规划不需要完美的世界模型，只需要一个在行动价值上足够准的模型。** 这在工程上是好消息——你不用预测每一个细节，你只需要预测"这样做会不会更好"。在很多现实场景中，"够用"比"完美"更容易达到，也更实用。

## 博导审稿

### 真正的创新点

论文的贡献层层递进，按重要性排列：

1. **隐空间规划**（最重要）：证明规划完全可以发生在学出来的抽象空间里，不需要回到原始观测。这是概念级的突破，打开了后续研究（如 Dreamer、EfficientZero）的大门。

2. **统一模型范式**（次重要）：第一次用一个算法同时征服了棋类游戏（离散、确定、稀疏奖励）和 Atari（视觉、随机、密集奖励）。之前的算法只能在其中一个领域工作。

3. **消除规则依赖**（结论性成果）：AlphaZero 系列的最后一块拼图被补上——连规则都不需要了。

### 可复现性

MuZero 的训练成本极高。论文报告使用了 1000 个 TPU（第一代）来训练围棋版本的 MuZero——这远超学术界大多数实验室的预算。后续的 EfficientZero（2021）和 MuZero Unplugged（2021）大幅降低了资源需求，但原始 MuZero 的可复现性对独立研究者来说是挑战，不是不可能但极其昂贵。

### 硬伤和局限

1. **样本效率仍然远不如人类**：MuZero 在 Atari 上需要约 5 千万帧才能超过人类——人类通常几分钟（几千帧）就能上手。数据效率是强化学习的长期瓶颈。

2. **只适用于"可模拟"的环境**：MuZero 需要能快速回放大量模拟——在围棋上每秒做上千次模拟。现实世界（如机器人控制）做不到这么快，实际的交互速度成了瓶颈。

3. **隐状态的可解释性为零**：动力学函数产生的隐状态是完全的黑盒。你无法知道"它在想什么"，无法调试它的推理过程。比起显式规则系统，这是从可解释性上的一次倒退。

4. **没有处理部分可观测性**：MuZero 假设环境是完全可观测的（或者通过 stacking 帧缓冲解决了时序信息）。对于真正部分可观测的环境（如扑克），它没有针对性设计。

5. **奖励结构假设简单**：MuZero 假设奖励信号是清晰可得的（得分、输赢）。在现实世界中，奖励往往延迟、稀疏或需要人为设计——这仍然是瓶颈。

### 后续演进

MuZero 开启了"基于模型的强化学习 + 隐空间规划"这个研究方向：
- **EfficientZero**（2021，Ye et al.）：通过增加 self-supervised 辅助损失（预测画面特征）大幅提高了样本效率，在 Atari 上以 1/50 的数据达到 MuZero 的水平
- **MuZero Unplugged**（2021）：让 MuZero 能从离线数据（非自我对弈产生）中学习
- **Dreamer V2/V3**（2021-2023）：基于隐空间的模型预测系列，在开放世界环境（Minecraft）中工作
- **Stochastic MuZero**（2022）：将 MuZero 扩展到随机环境

## 启发

### 如果你做研究

- **问对的问题比做好算法更重要**。MuZero 的问法不是"怎么把规划做得更好"，而是"规划必须建立在规则上吗"——这个问题的位移把整个领域向前推了一大步。
- **"够用就好"是一个被低估的设计原则**。学一个"在决策维度上够准"的模型比学一个"全知"的模型容易得多。在你自己的系统里，能不能砍掉对"完美"的执念，只保证在关键维度上的精度？

### 如果你是工程师

- **抽象层选对了，复杂性会自然消失**。MuZero 把需要在像素上做的推演压缩到了隐空间里，规划的计算复杂度就降下来了。你的系统里，有没有一个地方可以用"更好的抽象"替代"更强的计算"？
- **模拟器不是必需的，够好的预测模型也能做规划**。很多工程系统假设"需要完美的数字孪生才能做仿真规划"——MuZero 告诉你，也许不需要。

### 如果你是门外汉

这个故事最核心的那个画面很简单：**一个人走进一个完全陌生的世界，没拿到说明书。但他试了几次之后，脑子里就开始能"预演"了——"如果我这么做，大概会怎样"。** 这个故事之所以重要，是因为它暗示了：真正的智能不是背规则的能力，是在没有规则的地方长出规则、在没有模拟器的地方长出内心预演的能力。

MuZero 不是通用人工智能，但它可能是走向通用人工智能的路上一块关键的垫脚石——**因为它证明了智能体可以在未知中自己长出规划能力，而不需要别人先给它画好地图。**
