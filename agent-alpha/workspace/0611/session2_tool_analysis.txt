工具效率对比分析
================================================================================

Session 2中AI面临的工具选择：

┌─────────────────────────────────────────────────────────┐
│  浏览器 navigate + scroll + click + snapshot 完整流程    │
│  成本: ~6-8次工具调用/页（navigate + wait + snapshot）    │
│  收益: 能看到渲染后的页面，处理JS动态内容                  │
│  缺点: 慢，尤其是大量页面的场景                           │
├─────────────────────────────────────────────────────────┤
│  fetch 直接抓取网页内容                                  │
│  成本: 1次工具调用/页                                    │
│  收益: 快速获取结构化文本                                 │
│  缺点: 无法处理JS渲染，可能错过动态内容                    │
├─────────────────────────────────────────────────────────┤
│  browser_navigate 仅导航到搜索页                         │
│  成本: 1次工具调用 —— 但只作为"搜索入口"                   │
│  收益: 利用CDP用户已登录浏览器的Cookie/搜索权限            │
│  缺点: 只导航不抓内容的话，需要配合其他工具获取数据          │
└─────────────────────────────────────────────────────────┘
